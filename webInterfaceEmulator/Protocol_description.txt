Описание протокола.

Все данные передаются в формате json, так как планируется настройка
посредством web-интерфейса.

Сейчас данные по сети передаются в текстовом виде, в дальнейшем 
планируется переход на protocol buffer с передачей данных в бинарном виде.

Пример json объекта:
{
    "command": "write_config",
    
    "network": [
        {
            "interface" : {
                "device": "my_cool_device",
                "mode": "my_cool_mode"
            }
        }
    ]
}

Поле "command" содержит команду на выполнение.

Список команд:

1.	
    status - запрос (получить статус + дополнительную информацию о системе)
    ответ  - OK/NOT OK + дополнительная информация о системе
2.  
    reboot - запрос (перезагрузка системы)
    ответ - OK/NOT OK + перезагрузка через 5 секунд после посылки ответа
    
3.
    shutdown - запрос (выключение системы)
    ответ - OK/NOT OK + выключение через 5 секунд после посылки ответа
    
4.    
    read_config - запрос (чтение всей конфигурации системы)
    ответ - OK/NOT OK + конфигурация системы, либо описание кода ошибки
    
5.
    write_config - запрос (запись конфигурации) + json объект с конфигурацией
    ответ - OK/NOT OK, + описание кода ошибки
    

На каждый запрос Серверная сторона должна получать ответ.
Если ответа не последовало, то необходимо сделать повтор запроса.
